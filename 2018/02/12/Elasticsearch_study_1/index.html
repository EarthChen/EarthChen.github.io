Elasticsearch学习笔记 -- 1

学习笔记
  • Elasticsearch

ElasticSearch是一个基于Lucene的搜索服务器。它提供了一个分布式多用户能力的全文搜索引擎，基于RESTful web接口。Elasticsearch是用Java开发的，并作为Apache许可条款下的开放源码发布，是当前流行的企业级搜索引擎。设计用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。

具体的大家可以再去网上看看介绍。这里就不阐述了

elasticsearch基本操作

es的api基本格式为:http://:/<索引>/<类型>/<文档id>

使用http动词来操作数据

创建索引

  1. 非结构化创建索引
    PUT /people

people为索引名

  1. 结构化创建索引
    POST /book/novel/_mappings

执行创建结构化时，索引必须存在，即执行上述命令时，book索引必须存在

如果索引不存在，使用下面的方式

PUT /people

  • settings为当前索引的配置，比如切片数备份数
  • es支持||代表当前属性支持多种格式

数据插入

  1. 指定文档id插入

PUT /people/man/1

  1. 自动生成id插入

POST /people/man

数据修改

  1. 直接修改

POST /people/man/1/_update

  1. 使用脚本

POST /people/man/1/_update

数据删除

  1. 删除指定的文档

DELETE /people/man/1

  1. 删除指定索引

DELETE /people

基本查询

查询结构为

简单查询
  1. 根据id查询数据

GET /book/novel/2

条件查询
  1. 查询所有数据

POST /book/_search

默认10条数据

  1. 指定从第几条返回，返回多少条数据

POST /book/_search

  1. 查询关键词

    • 查询title中含有title的数据

      POST /book/_search

    • 增加排序规则（按照publish_date的降序排列）

      POST /book/_search

聚合查询
  1. 按照指定字段聚合查询

POST /book/_search

  1. 按照指定字段聚合计算

POST /book/_search

高级查询

子条件查询(特定字段查询所指特定值)
  1. Query context

(在查询过程中,除了判断文档是否满足查询条件,es还会计算一个_score,来标示匹配的程度，为了判断目标文档和查询条件有多好)

  • 全文本查询(针对文本类型数据)

    1. 模糊匹配(分词匹配)match

      查询title中含有elasticsearch或者入门的数据

      POST /book/_search

    2. 不分词匹配(将查询内容作为一个整体)match_phrase

      查询title中含有elasticsearch学习的数据

      POST /book/_search

    3. 对多个字段的匹配查询multi_match

      查询title或者author字段中含有elasticsearch的数据

      POST /book/_search

    4. 语法查询query_string

      查询含有elasticsearch和学习或者java的数据

      POST /book/_search

    5. 指定字段语法查询query_string

      查询字段title或者author中含有elasticsearch和学习或者java或者aaa的数据

      POST /book/_search

}

  • 字段级别查询(针对结构化数据,数字，日期等)term

    1. 对指定字段的精确查询

      1. 查询word_count等于1000的的数据

        POST /book/_search

      2. 查询word_count在范围1000到3000内的数据

        POST /book/_search

  1. Filter context

    (在查询过程中，只判断该文档是否满足条件，只有yes或者no，并且会缓存结果)

    查询出word_count等于1000的数据

复合条件查询(以一定的逻辑组合子条件查询)

  1. 固定分数查询

    查询所有索引中title为java

    POST /_search

  2. 布尔查询

    1. 查询所有索引中author为aaa或title为elasticsearch （should）

      POST /_search

    2. 查询所有索引中author含有aaa,title含有java的数据 (must)

      POST /_search

    3. 查询所有索引中author含有aaa,title含有java，并且过滤word_count为1000的数据 (must)

      POST /_search

    4. 查询所有索引中author不含有aaa,并且title不含有java的数据 (must_not)

      POST /_search
